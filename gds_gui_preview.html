◈ 使用說明

▸ 掃描工作流程

步驟操作說明
1掃描 GDS上傳 GDS 檔案，自動使用推測規則分析層配置。
掃描完成後不自動執行分析，由用戶自行選擇操作。
2修改與儲存
(可選)		掃描後可在表格「修改」欄位手動調整層的用途。
按「儲存 Mapping JSON」確認，檔案存於 output/ 目錄。
3加載 Mapping JSON
(可選)		若需套用之前保存的層映射，點擊「加載 Mapping JSON」按鈕。
系統自動查找 output/ 和 input/ 目錄中的 JSON 檔。
4執行分析點擊「Bump / Amark 分析」加載 Bump Type 分類。
點擊「輸出 PAD Excel」生成最終 Excel 報表。
工作流程 2、3 可選，直接執行步驟 4 即可；若需修改層對應規則才進行步驟 2、3。

▸ Layer 對應規則

精確強制對應（Layer : DType，優先級最高）

LayerDType強制用途說明
1909chip_size預設晶片邊界層（優先級最高）

預設 Layer 對應（DType 固定）

用途LayerDType說明
chip_size1909晶片邊界矩形（優先級最高）
amark205224Alignment Mark 外框矩形
bump224, 2250Bump / PAD 幾何圖形（含 #POLY 一律強制識別）
bump_text_number224, 2250Bump 序號標注（含 #TEXT 一律強制識別）
bump_text_name211224Bump 信號名稱（如 VDD、RX[0]）
預設對應優先級最高。若需要自訂，掃描後在表格「修改」欄位手動調整，再按「儲存 Mapping JSON」。
特例：Layer 224, 225 DType 0 的強制識別不受 mapping JSON 影響：
  • 含 #POLY（polygon）一律為 BUMP/PAD
  • 含 #TEXT 一律為 Bump 序號標注

幾何自動推測規則（polygon，依優先順序）

條件推測用途
polygon 數 ≤ 4，平均面積 > 100,000 µm²chip_size
polygon 數 ≤ 8，最大頂點數 ≥ 8（十字形），寬/高 ≤ 150 µmamark
polygon 數 ≤ 20，最小頂點數 = 3（三角形），寬/高 ≤ 150 µmamark
polygon 數 > 4，平均面積 < 5,000 µm²，長寬比 0.5–2.0bump
其他unknown

文字內容自動推測規則（text）

文字特徵推測用途
amark layer（如 205）的附屬文字amark_text（自動歸類）
同 layer polygon 平均面積 > 10,000 µm²（大型外框附屬）unknown（忽略）
含「NO」（大小寫不拘，如 NO3501）bump_text_number
含英文字母（如 RX[9]、VDDD）bump_text_name
純數字bump_text_number

PAD 分類規則（Excel 輸出）

條件分類說明
密集單排（同 Y 行 > 10 個，±0.5 µm 容差）+ 晶片下半部
→ 固定 1 排，靠近 IC 下半部 edge		INPUT密集排列，僅 1 排
密集單排（同 Y 行 > 10 個，±0.5 µm 容差）+ INPUT 上緣以上
→ 可為 1 排以上，靠近 IC 上半部 edge		OUTPUT密集排列，1 排以上
其他（邊緣稀疏 / 角落稀疏 / IC 中間稀疏）DUMMY雜散 PAD，不符合 INPUT/OUTPUT 分類

Excel 輸出欄位說明

符號說明計算方式
A1-A3INPUT PAD 尺寸按 W×H 統計，按數量排序
A4INPUT to IC 下方邊min(INPUT PAD 底部到 chip 下邊距離)
A5INPUT to IC 側邊min(INPUT PAD 左/右邊到 chip 側邊距離)
B1-B2OUTPUT PAD 尺寸按 W×H 統計，按數量排序
B3OUTPUT to IC 上方邊min(OUTPUT PAD 頂部到 chip 上邊距離)
B4OUTPUT to IC 側邊min(OUTPUT PAD 左/右邊到 chip 側邊距離)
B5OUTPUT to INPUT 間距OUTPUT 最下點 - INPUT 最上點（只用靠近 chip 中心的 OUTPUT）
C1-C2DUMMY PAD 尺寸按 W×H 統計，按數量排序
C3DUMMY to IC 上方邊min(DUMMY PAD 頂部到 chip 上邊距離)
C4DUMMY to IC 側邊min(DUMMY PAD 左/右邊到 chip 側邊距離)
所有距離計算一律使用 real（實際最小值），嚴禁使用 avg（平均值）。

Layer 用途說明

用途說明
bumpBump / PAD 幾何圖形，用於計算座標、尺寸、PAD 清單
amarkAlignment Mark 外框矩形，定位晶片角落標記
amark_textAlignment Mark 附屬文字，不列入 PAD 清單
bump_text_nameBump 信號名稱標注（如 VDD、RX[0]）
bump_text_numberBump 序號標注（如 NO0001）
chip_size晶片邊界矩形，用於計算 Chip W × H
origin_mark晶片 (0, 0) 原點標記，不參與分析
unknown未識別用途，可在表格中手動修改
掃描後可在表格「修改」欄位手動調整各 layer 的用途，再按「儲存 Mapping JSON」儲存確認結果。
後續「Bump / Amark 分析」與「輸出 PAD Excel」均以已儲存的 JSON 為準。
